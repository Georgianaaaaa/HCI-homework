Introducere

In order to achieve my BSc title, I chose to make a software application called Smart Parking.

This is an application that can easily find the closest parking space available from the point the user is find, so this no longer has to waste time in the city looking for one.

With the help of a map that contains all the parking lots in the city, the application finds the current location of the user and based on this it looks for the nearest point where the user can park the car.

Structure

The app has 3 main components: the start page, the user page and the page that manage the effective parking of the car.

3.1 The start page

The start page contains the next sections: about, contact, help and log in.

The idea here is to make this page very intuitive and easy to use. If the user has questions regarding the app, what exactly it is and what it does, how it can be used or it wants to contact the admin/s, it can access the sections about, contact or help.

The log in section is the most important here. The user must to be logged in in order to use this app!!! If it has already an account, it needs to sign in using the username and the password and if it has no account, it can create a new one.

The user page

Once logged in, the user can use the application for finding a parking space for the car by clicking the button park which leads to a page where more information about the parking are required.

The parking page

This page contains multiple fields that need to be filled in with different information such as the city in which you want to park your car. After the desired city is selected, the application will find the closest parking space available and after that you need to:

  • insert the number plate of your car
  • insert the duration of parking
  • select a method for paying (if it is required)
